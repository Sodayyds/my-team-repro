文章开发.EXE 需求文档

1. 项目开发目标
本工具面向论文、自媒体文章自动化产出，搭建一站式图文稿件生成+投稿自动化程序，降低人工重复操作成本。
程序需自动完成论文撰写、AI配图生成、作者与期刊信息打包、多平台自动投稿；最终输出满足5000字以上、图文配套、参考文献≥40篇的完整标准化文稿。

2. 核心功能需求

2.1 调度分发模块
程序整体拆分为四大并行业务模块统一调度：写文章模块、画图模块、作者信息配置模块、自媒体投稿模块，多任务并行执行。

2.2 文章撰写模块
完整执行链路：选题定位 → 检索收集不少于40篇参考文献 → 自动生成初稿 → AI优化润色 → 模拟审稿校对 → 定稿输出完整文稿。

2.3 AI配图生成模块
基于行业优质配图总结生成标准化提示词，通过多轮对话迭代生成和文稿内容匹配的插图，自动插入文档对应位置。

2.4 作者/期刊信息管理模块
维护本地作者信息、期刊投稿要求数据表，自动整理全部资料，打包生成标准投稿压缩包。

2.5 自动投稿模块
AI驱动完成平台账号登录、自动人机验证、稿件一键发布、返修意见自动识别与修改重投。

2.6 统一输出功能
整合文字、配图、参考文献、作者资料，输出合规完整成品文章文件。

3. 非功能需求
1. 基于Windows打包为EXE程序，开箱即用，无需复杂环境配置；
2. 模块解耦，单模块可单独迭代优化，不影响整体程序运行；
3. 所有配置、文稿、图片、打包文件统一本地留存，可追溯；
4. 支持多人协作开发，使用Git分支管理迭代，变更可记录、可回滚。

4. 开发协作需求（GitHub仓库配套）
1. 仓库统一存放项目需求、架构、开发记录文档；
2. 使用Issues记录各模块开发任务、优化需求；
3. 采用多分支开发，新功能分支开发完成后通过PR合并至主分支；
4. 每次代码/文档变更均保留提交记录，方便团队查看迭代过程。
